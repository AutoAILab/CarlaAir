# Set-up Project 


## Tasks
1. Fill in `ticket` workflow, located `.agents/workflows/ticket.md`
    - Fully read the ticket
    - Fully read through relevant documentation and files in the codebase
    - Fill in ticket with relevant information and project guidelines
    - Ticket should have following structure:
        - Title
        - Description
        - Context
        - Constraints & style
        - Acceptance criteria
        - Execution plan
    - Ask user for feedback and iterate until user is satisfied
    - If user wants to proceed, create implementation plan and start ticket
    - If user does not want to proceed, ask user what they want to change
2. Fill in `complete` workflow, located `.agents/workflows/complete.md`
    - Verify that the ticket has been completed by examining ticket requirements
    - Commit changes via git
2. Fill in `rules` workflow, located `.agents/rules/`
